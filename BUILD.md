# Chia Market Maker — Windows Build Guide

## Prerequisites

- Python 3.10 or later (same version used for development)
- All runtime dependencies installed (run the bot at least once in dev mode first)
- PyInstaller (the build script installs it automatically if missing)

```
pip install pywebview pystray plyer Pillow flask requests python-dotenv --break-system-packages
```

## How to Build

```bash
python build.py
```

That's it. The script:
1. Checks PyInstaller is installed and installs it if not
2. Cleans `dist/` and `build/` from any previous run
3. Runs PyInstaller with `build_windows.spec`
4. Copies `.env.example` into the output folder
5. Verifies the exe and HTML assets are present

To rebuild without cleaning (faster iteration):
```bash
python build.py --no-clean
```

## Configuring the App

1. Navigate to `dist/ChiaMarketMaker/`
2. Copy `.env.example` to `.env`
3. Open `.env` in a text editor and fill in your values (at minimum: wallet credentials and `CAT_ASSET_ID`)
4. Double-click `ChiaMarketMaker.exe`

The `.env` file is never bundled — it stays on your machine and holds your wallet credentials.

## Distributing

Copy the entire `dist/ChiaMarketMaker/` folder to the target machine. The folder is self-contained. No Python installation required on the target machine.

The user creates their own `.env` file alongside the exe.

## Known Issues and Limitations

### WebView2 / Edge required
PyWebView on Windows uses Microsoft Edge WebView2. This ships pre-installed on Windows 11 and is auto-installed on Windows 10 via Windows Update. If the target machine is very old or has WebView2 disabled, the app will fail to launch. Users can download WebView2 from: https://developer.microsoft.com/en-us/microsoft-edge/webview2/

### Console window
`ChiaMarketMaker.exe` is built as a GUI executable (`console=False`) so no terminal window appears. Crash details are written to `crash.log` in the same directory as the exe.

### Antivirus false positives
PyInstaller bundles are sometimes flagged by antivirus software. This is a known PyInstaller issue. If flagged, users need to add the folder to their AV exclusions. The bundle contains no network code beyond what the Python trading bot itself uses.

### splash.exe (P2P node)
`splash.exe` is bundled if it exists in the project root at build time. If it was absent, users who enable `SPLASH_ENABLED=true` need to download it separately from https://github.com/dexie-space/splash/releases and place it in the same folder as `ChiaMarketMaker.exe`.

### Database and logs
The bot creates `bot.db`, `crash.log`, and `bot_superlog_*.log` files at runtime. These are written to the current working directory when the exe is launched — typically the same folder as the exe. Do not delete them while the bot is running.

### .env hot-reload
The GUI's Settings panel writes changes back to the `.env` file. This works the same as in dev mode — the file must exist and be writable.

### Wallet certificate paths
`SAGE_CERT_PATH`, `SAGE_KEY_PATH`, `CHIA_WALLET_CERT`, and `CHIA_WALLET_KEY` must be absolute paths on the target machine. They point to certificate files generated by the wallet software and are never bundled.
