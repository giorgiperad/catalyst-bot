# Chia Market Maker — Windows Build Guide

## Prerequisites

- Python 3.10 or later (same version used for development)
- All runtime dependencies installed (run the bot at least once in dev mode first)
- PyInstaller (the build script installs it automatically if missing)

```
pip install pywebview pystray plyer Pillow flask requests python-dotenv --break-system-packages
```

## How to Build

```bash
python build.py
```

That's it. The script:
1. Checks PyInstaller is installed and installs it if not
2. Cleans `dist/` and `build/` from any previous run
3. Runs PyInstaller with `build_windows.spec`
4. Copies `.env.example` into the output folder
5. Verifies the exe and HTML assets are present

To rebuild without cleaning (faster iteration):
```bash
python build.py --no-clean
```

## Configuring the App

1. Navigate to `dist/ChiaMarketMaker/`
2. Copy `.env.example` to `.env`
3. Open `.env` in a text editor and fill in your values (at minimum: wallet credentials and `CAT_ASSET_ID`)
4. Double-click `ChiaMarketMaker.exe`

The `.env` file is never bundled — it stays on your machine and holds your wallet credentials.

## Distributing

Copy the entire `dist/ChiaMarketMaker/` folder to the target machine. The folder is self-contained. No Python installation required on the target machine.

The user creates their own `.env` file alongside the exe.

## Known Issues and Limitations

### WebView2 / Edge required
PyWebView on Windows uses Microsoft Edge WebView2. This ships pre-installed on Windows 11 and is auto-installed on Windows 10 via Windows Update. If the target machine is very old or has WebView2 disabled, the app will fail to launch. Users can download WebView2 from: https://developer.microsoft.com/en-us/microsoft-edge/webview2/

### Console window
`ChiaMarketMaker.exe` is built as a GUI executable (`console=False`) so no terminal window appears. Crash details are written to `crash.log` in the same directory as the exe.

### Antivirus false positives
PyInstaller bundles are sometimes flagged by antivirus software. This is a known PyInstaller issue. If flagged, users need to add the folder to their AV exclusions. The bundle contains no network code beyond what the Python trading bot itself uses.

### splash.exe (P2P node)
`splash.exe` is bundled if it exists in the project root at build time. If it was absent, users who enable `SPLASH_ENABLED=true` need to download it separately from https://github.com/dexie-space/splash/releases and place it in the same folder as `ChiaMarketMaker.exe`.

### Database and logs
The bot creates `bot.db`, `crash.log`, `bot_superlog_*.log`, and the `.env` file in a **per-user data directory**, NOT next to the exe. This lets the app be installed to read-only locations like `C:\Program Files\`.

| OS | Data directory |
|----|----------------|
| Windows | `%APPDATA%\ChiaMarketMaker\` (e.g. `C:\Users\you\AppData\Roaming\ChiaMarketMaker\`) |
| macOS | `~/Library/Application Support/ChiaMarketMaker/` |
| Linux | `$XDG_DATA_HOME/ChiaMarketMaker/` (defaults to `~/.local/share/ChiaMarketMaker/`) |

Override with the `CMM_DATA_DIR` environment variable (useful for portable installs).

On first launch, if a legacy `.env` / `bot.db` / log files are found next to the exe (dev layout), they are automatically copied into the data dir. A `.migration_complete` marker is written so the migration only runs once.

### .env hot-reload
The GUI's Settings panel writes changes back to the `.env` file in the data dir. On first launch the file is seeded from `.env.example` (bundled next to the exe) if no `.env` exists yet.

### Wallet certificate paths
`SAGE_CERT_PATH`, `SAGE_KEY_PATH`, `CHIA_WALLET_CERT`, and `CHIA_WALLET_KEY` must be absolute paths on the target machine. They point to certificate files generated by the wallet software and are never bundled.

## Code Signing (CRITICAL for distribution)

An **unsigned** `ChiaMarketMaker.exe` will trigger a red "Windows protected your PC" SmartScreen warning on first launch. Most non-technical users will close that dialog and never launch the app. If you are distributing this to end users, **code signing is not optional**.

### Get a certificate

Two tiers of Authenticode cert:

| Type | Cost (approx.) | SmartScreen behaviour |
|------|----------------|------------------------|
| **OV** (Organization Validation) | $150–$400/yr | Warning persists until Microsoft reputation builds (weeks to months) |
| **EV** (Extended Validation) | $300–$700/yr | **Instant reputation** — no SmartScreen warning from day one |

For a paid consumer product, buy an EV cert. Vendors: Sectigo, DigiCert, SSL.com, GlobalSign. EV certs ship on a hardware USB token that must be plugged in every time you sign.

### Sign with signtool

`signtool.exe` ships with the Windows SDK. Typical path:
`C:\Program Files (x86)\Windows Kits\10\bin\10.0.22621.0\x64\signtool.exe`

**Important: sign every bundled binary, not just the main exe.** SmartScreen checks signatures on every child process the main exe launches. `splash.exe` is one of those child processes, so it must be signed with the same cert. **Sign child binaries first, then the parent.**

```bat
:: 1. Sign splash.exe first
signtool sign ^
    /fd sha256 ^
    /td sha256 ^
    /tr http://timestamp.sectigo.com ^
    /a ^
    "dist\ChiaMarketMaker\splash.exe"

:: 2. Then sign the main exe
signtool sign ^
    /fd sha256 ^
    /td sha256 ^
    /tr http://timestamp.sectigo.com ^
    /a ^
    "dist\ChiaMarketMaker\ChiaMarketMaker.exe"
```

Flags:
- `/fd sha256` — SHA-256 for the file digest
- `/td sha256` — SHA-256 for the timestamp
- `/tr http://timestamp.sectigo.com` — RFC 3161 timestamp server. Alternatives: `http://timestamp.digicert.com`, `http://timestamp.globalsign.com/tsa/r6advanced1`
- `/a` — auto-pick the best cert from the Windows cert store. For an EV USB token you may need `/n "Your Company Name"` or `/sha1 <thumbprint>` instead.

### Verify the signature

```bat
signtool verify /pa /v "dist\ChiaMarketMaker\splash.exe"
signtool verify /pa /v "dist\ChiaMarketMaker\ChiaMarketMaker.exe"
```

Look for `Successfully verified` and a populated certificate chain.

### Sign the installer too

If you wrap the `dist\ChiaMarketMaker\` folder in an installer (Inno Setup, NSIS, MSI), **sign the installer exe as well** using the same command. Users run the installer first, so SmartScreen will flag an unsigned installer even if the inner exe is signed.

### EV token on CI

Hardware tokens don't play well with unattended CI. Options:
1. Sectigo / DigiCert offer cloud signing services (eSigner, KeyLocker, etc.) — budget an extra $200–$400/yr
2. Self-host a Windows runner with the token plugged in permanently

### Troubleshooting

- **"No certificates were found that met all the given criteria"** — `/a` searches your personal cert store. For a token cert, use `/n "Exact Subject"` or `/sha1 <thumbprint>`. Make sure the token is plugged in and unlocked.
- **SmartScreen still warns after OV signing** — Expected. Reputation builds over time. Submit to Microsoft Defender for faster review, or upgrade to EV.
- **Child process signature invalidated** — You signed the parent exe before the child. Re-sign both, child first.

### macOS notes

For a Mac distribution build, enroll in the Apple Developer Program ($99/yr), get a **Developer ID Application** cert in Xcode, then:

```bash
codesign --deep --force --verify --verbose \
    --sign "Developer ID Application: Your Name (TEAMID)" \
    --options runtime \
    "dist/ChiaMarketMaker.app"

# Notarize
ditto -c -k --keepParent "dist/ChiaMarketMaker.app" "ChiaMarketMaker.zip"
xcrun notarytool submit "ChiaMarketMaker.zip" \
    --apple-id "you@example.com" --team-id "TEAMID" \
    --password "app-specific-password" --wait

# Staple the approval
xcrun stapler staple "dist/ChiaMarketMaker.app"
```

Use an **app-specific password** (generate at appleid.apple.com), never your real Apple ID password.
